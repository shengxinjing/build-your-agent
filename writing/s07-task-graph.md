# s07：任务不只是一张清单，它应该落到磁盘

s03 的计划工具解决了“先想再做”。

但它还是有一个大缺点：

**计划只活在内存里。**

一旦会话重启，或者上下文压缩过头，任务关系就容易丢。

所以 s07 要做升级：

- 任务写入磁盘
- 每个任务有状态
- 任务之间能表达依赖

这就从“计划”走到了“任务图”。

## 为什么一定要落盘

因为很多真实任务天然就是跨轮次、跨上下文、甚至跨 agent 的。

比如：

- 先解析
- 再生成
- 最后测试

这里面“测试”不会因为一次聊天结束就失去意义。

所以任务系统必须比当前会话活得更久。

## 这节的最小实现

我们用最简单的方式：

- 一个任务一个 JSON 文件
- 存在 `tasks/` 目录里
- `blockedBy` 表示依赖
- `completed` 时自动解除后续阻塞

这已经足够给后面的：

- 后台任务
- 多 agent 协作
- worktree 隔离

打基础。

## 运行

```bash
node writing/s07-task-graph.js "Create a small dependency graph"
```

自测：

```bash
node writing/s07-task-graph.js --self-test
```

## 这一节学会什么

- 任务系统和计划工具不是一回事
- 任务要持久化
- 依赖一旦明确，后面并行和协作才有依据
