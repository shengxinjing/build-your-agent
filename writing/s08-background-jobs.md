# s08：慢操作丢后台，agent 继续往前走

到 s07 为止，agent 还是同步思维。

也就是：它一旦跑一个慢操作，就只能等。

这在现实里会很浪费：

- 跑测试可能要几十秒
- 打包可能要几分钟
- 拉远端数据也可能很慢

所以 s08 的目标很明确：

**把慢操作放后台，完成后再把通知注入回来。**

## 这一节的关键不是并发语法

关键是两个设计点：

1. 后台任务有自己的状态
2. 主 loop 会收到“任务完成”的通知

也就是说，主 agent 不需要一直堵在那里等。

它可以：

- 启动后台 job
- 继续处理别的事
- 过一会儿收到完成通知

## 代码怎么做

这一节我们加一个 `start_background_job` 工具。

真正的 job 由 `BackgroundJobs` 管理器托管。

每一轮 loop 结束后，我们检查一次：

- 有没有新完成的 job
- 如果有，就拼成一条新消息塞回上下文

这就是“通知注入”。

## 运行

```bash
node writing/s08-background-jobs.js "Start a slow task and keep working"
```

自测：

```bash
node writing/s08-background-jobs.js --self-test
```

## 这一节学会什么

- 后台执行不是单纯为了快
- 更重要的是：主 agent 的思路不被阻塞
- 后台任务完成时，要重新回到消息流里
