# s03：没有计划，agent 很容易漂

到这一节，agent 已经能做事了。

但它还不稳定。

为什么？因为模型一旦急着开工，就容易：

- 漏步骤
- 先做后想
- 中途忘了自己做到哪

所以我们先补一个最简单，但非常关键的工具：

**`update_plan`**

## 为什么计划要做成工具

很多人第一反应是：那我在 prompt 里说一句“先计划再执行”不就好了？

不够。

因为“说过”不等于“留下状态”。

工具的价值在于：

1. 计划会变成结构化数据
2. 每一步有状态
3. agent 可以自己更新进度
4. 之后你还可以把它持久化到磁盘

这一节我们先做最小版：计划只存在内存里。

## 计划工具长什么样

它接受一个数组：

```json
[
  { "id": "1", "text": "列计划", "status": "completed" },
  { "id": "2", "text": "写文件", "status": "in_progress" }
]
```

我们故意限制只有一个 `in_progress`，这样状态会更清楚。

## 运行

```bash
node writing/s03-plan-first.js "Make a small plan and save it to outline.md"
```

自测：

```bash
node writing/s03-plan-first.js --self-test
```

## 这一节学会什么

- “先计划再执行”最好变成工具，而不是只放在 prompt 里
- 计划一旦结构化，后面就能继续升级成任务图
- s03 和 s07 的关系是：前者是内存计划，后者是磁盘任务系统
