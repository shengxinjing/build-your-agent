# s12：每个任务一个目录，互不干扰

现在我们已经有了：

- agent loop
- tools
- plan
- subagent
- 按需知识
- 上下文压缩
- 任务图
- 后台任务
- 队友和协议
- 自组织认领

最后一个现实问题是：

**大家都在同一个目录里改东西，迟早互相踩。**

所以 s12 做的事是：

**任务管目标，worktree 管目录，两者绑定。**

## 为什么任务和目录要分开

任务系统回答的是：

- 要做什么
- 谁在做
- 依赖是什么

worktree 系统回答的是：

- 在哪里做
- 对应哪条分支
- 什么时候回收

这两个东西不能混在一起，但必须绑上。

## 这一节做了什么

代码里会：

1. 初始化一个 demo git 仓库
2. 创建两个任务
3. 为两个任务各建一个 worktree
4. 把 worktree 名字写回任务
5. 删除一个 worktree，并自动把对应任务标记为完成

这一步结束后，你已经得到一个非常像真正 code agent 的最小骨架。

## 运行

```bash
node writing/s12-worktree-isolation.js --self-test
```
