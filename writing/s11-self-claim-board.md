# s11：队友自己扫任务板，看到活就认领

到 s10 为止，团队已经能稳定通信了。

但分工还是偏“领导驱动”：

- lead 先想
- lead 再发请求
- 队友再执行

这当然能跑，但效率一般。

更进一步的方式是：

**队友自己看任务板，发现可做任务后主动认领。**

## 为什么这一步重要

因为只要任务依赖关系已经清楚，很多事根本不需要领导逐个指挥。

比如：

- task A 做完
- task B 和 task C 解锁
- 谁空着谁去拿

这就是自组织协作。

## 这节的重点不在 prompt

这一步更像“控制平面设计”。

我们已经有了：

- 持久化任务图
- 明确的状态
- 明确的依赖

那接下来只要给 worker 一个规则：

**反复扫描 ready tasks，抢到就做，做完就回写状态。**

## 运行

```bash
node writing/s11-self-claim-board.js --self-test
```

## 这一节学会什么

- 自组织的前提不是更聪明的 prompt
- 而是更明确的共享状态
- 任务板一旦可靠，领导就不需要逐个分配
