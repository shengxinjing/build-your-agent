# s05：知识按需加载，不要一开始全塞进去

到这里，agent 已经会：

- 调工具
- 列计划
- 派 subagent

但还有一个很常见的问题：

**你到底要不要把所有知识都先塞进 system prompt？**

答案通常是：**不要。**

## 为什么不要提前全塞

因为大部分任务只会用到一小部分知识。

如果你把所有文档、规范、接口说明都提前塞进去，会有三个坏处：

1. 上下文很快变大
2. 模型注意力会被无关信息分散
3. 你很难知道它到底用了哪一份知识

更好的做法是：

- 先告诉模型“有哪些知识可用”
- 真需要时，再通过工具加载
- 把加载结果作为 tool result 注入消息流

## 这一节怎么实现

我们增加一个 `load_knowledge` 工具。

它不负责思考，只负责：

- 根据主题名找文件
- 把知识文本读出来
- 作为工具结果返回

主 agent 拿到这份知识后，再继续做判断。

## 运行

```bash
node writing/s05-load-knowledge.js "Read the release rule and write a short checklist"
```

自测：

```bash
node writing/s05-load-knowledge.js --self-test
```

## 这一节学会什么

- 知识应该按需注入
- `tool_result` 不只是放命令输出，也可以放文档片段
- 这一步能明显降低 prompt 污染
